# Prism-Learn Feedback Integration

## Overview

Prism supports receiving feedback from prism-learn—an external service that analyzes aggregate Ansible role metadata and provides recommendations to improve documentation quality and consistency.

Feedback can be provided via:
- **Local file**: `--feedback-from-learn /path/to/feedback.json`
- **Live API**: `--feedback-from-learn https://prism-learn.example.com/api/feedback?role=my_role`

This is part of a planned two-phase rollout:
- **Phase 1** (current): Infrastructure and schema definition
- **Phase 2** (future): prism-learn generates feedback; prism applies recommendations

## Using Feedback

### Local File

```bash
# Generate documentation using local feedback.json
prism role /path/to/role --feedback-from-learn /path/to/feedback.json
```

### Live API Endpoint

```bash
# Generate documentation using live prism-learn API feedback
prism role /path/to/role --feedback-from-learn https://learn.example.com/api/feedback?role=my_role

# With basic auth (if required):
prism role /path/to/role --feedback-from-learn 'https://user:pass@learn.example.com/api/feedback'
```

If feedback recommends certain audits, they will be enabled/disabled accordingly, overriding CLI flags.

### Error Handling

- **File not found**: Exits with code 1, prints error message
- **API unreachable**: Exits with code 1, prints network error
- **Malformed JSON**: Exits with code 1, prints JSON parse error
- **Invalid payload**: Exits with code 1, prints validation error

Example (with timeout):
```bash
# If prism-learn is slow, feedback request will timeout after 10 seconds
# and prism will exit with an error rather than hanging
```

## Feedback Schema

The feedback.json file follows this TypeScript-style schema:

```typescript
interface FeedbackPayload {
  version: string;
  // Schema version for forward compatibility (e.g., "1.0")

  generated_at: string;
  // ISO 8601 timestamp when feedback was generated by prism-learn

  analysis_window: {
    date_start?: string;
    date_end?: string;
    roles_analyzed?: number;
    // Metadata about the analysis window
  };

  defect_categories: {
    [categoryName: string]: {
      severity: string;        // "high", "medium", "low"
      count: number;           // How many roles exhibit this defect
      description: string;     // Human-readable summary
      percentage?: number;     // Percentage of analyzed roles
    };
  };
  // Aggregate statistics of defect patterns found

  recommendations: [
    {
      type: string;
      // Recommendation category, e.g.: "check_collection_compliance"

      display: boolean;
      // Whether to display/enable this audit in prism output

      reason: string;
      // Why prism-learn recommends this

      severity?: string;
      // "high", "medium", "low"

      additional_data?: {
        [key: string]: any;
      };
      // Optional extra context for implementation
    }
  ];
  // Ordered list of recommended scanner behavior changes

  summary: string;
  // Human-readable executive summary of findings
}
```

### Minimal Example

```json
{
  "version": "1.0",
  "generated_at": "2026-03-19T18:00:00Z",
  "defect_categories": {
    "missing_collection_declarations": {
      "severity": "high",
      "count": 247,
      "description": "Roles using external collections without declaring them",
      "percentage": 68.4
    }
  },
  "recommendations": [
    {
      "type": "check_collection_compliance",
      "display": true,
      "reason": "Your collection usage rate is 68% higher than baseline",
      "severity": "high"
    }
  ],
  "summary": "Most roles in this batch need collection compliance fixes."
}
```

## Integration Points

### CLI Parameter

All prism subcommands support feedback:
- `prism role --feedback-from-learn feedback.json`
- `prism collection --feedback-from-learn feedback.json`
- `prism repo --feedback-from-learn feedback.json`

### Error Handling

If feedback.json is invalid:
- Malformed JSON → error message and exit code 1
- Missing required fields → silently ignored (backward compatible)
- File not found → error message and exit code 1

### Python API

```python
from prism.feedback import load_feedback, apply_feedback_recommendations

# Load feedback from file
feedback = load_feedback("/path/to/feedback.json")

# Load feedback from API
feedback = load_feedback("https://learn.example.com/api/feedback?role=my_role")

# Apply recommendations
settings = apply_feedback_recommendations(
    feedback,
    include_collection_checks=False  # Current CLI flag value
)

# settings["include_collection_checks"] may now be overridden by API response
# settings["recommendations_applied"] lists what was changed
```

Supported sources in `load_feedback()`:
- `None` — skip loading (returns None)
- `/path/to/file.json` — local file
- `https://api.example.com/endpoint` — remote API (follows HTTP redirects)
- `http://...` — also supported (not recommended in production)

## Future Enhancements (Phase 2+)

Planned feedback-driven features:

1. **Variable Detection Hints**
   - prism-learn identifies common variable patterns
   - Recommends stricter/looser variable type inference
   - Recommendation type: `set_variable_confidence_threshold`

2. **Task Complexity Analysis**
   - prism-learn detects overly complex task sequences
   - Recommends expanded task documentation
   - Recommendation type: `expand_task_documentation`

3. **Collection Plugin Reporting**
   - prism-learn tracks plugin capabilities vs. role exports
   - Recommends collection plugin extraction and documentation
   - Recommendation type: `document_collection_plugins`

4. **Role Quality Scoring**
   - prism-learn assigns quality scores based on aggregate defects
   - Prism includes score badges/summaries in output
   - Recommendation type: `include_quality_score`

5. **Batch Feedback Processing**
   - prism-learn analyzes entire collection roots
   - Provides role-specific feedback
   - `--feedback-from-learn` can be a directory; prism selects role-specific files

## Development Notes

### For prism-learn Developers

To integrate with prism:

1. Generate feedback.json matching the schema above
2. Set `recommendations[].display` based on your analysis
3. Call prism with `--feedback-from-learn /path/to/feedback.json`
4. Monitor exit codes and logs to detect recommendation success

### For prism Contributors

The feedback system is designed to be extensible. To add a new recommendation type:

1. Update `prism/feedback.py` with the new type definition
2. Implement logic in `apply_feedback_recommendations()`
3. Document the new type in this file and in `--help`
4. Add tests

See `src/prism/feedback.py` for implementation details and TypedDict schemas.
